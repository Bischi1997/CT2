Synchronous Bus Timing Options

  • Master und Slaves benutzen eine gemeinsame Clock.
  • Clock steuern den Bustransfer auf beiden Seiten.

Asynchronous Bus Timing Options

  • Slaves haben keinen Zugriff auf die Clock des Masters.
  • Control signalisiert carry timing information um eine Synchronisation zu erlauben.

Mehrere Devices benutzen die gleiche Data Line

Wenn ein Device eine logische 1 und ein anderes Device eine logische 0 sendet, dann führt dies zu einem Kurzschluss.

Aus diesem Grund definiert die CPU wer den Daten Bus zu einer gegebenen Zeit benutzen darf.

  • Write → CPU fährt den Bus → Alle Slaves sind vom Bus disconnected
  • Read → CPU vom Bus disconnected → Einzelner Slave fährt den Bus mit Values auf einer gegeben Adresse, alle andere Slaves sind disconnected

Buszugriffe (Synchronous Bus)

Block Diagramm

  • Adress Linien A[31:0]
  • Daten Linien D[31:0]
  • CLK → Clocksignal
  • NE → Not Enable: zeigt start oder ende eines zyklus (cycle), active-low
  • NWE → Not Write Enable: active-low

Active Low bedeutet wenn Value = 0 dann ist der Output enabled. Wenn Value = 1 dann ist der Output disabled. Ist eine Art Inverse.

General Purpose I/O

Mode register (GPIOx_MODER)

Wenn Eingabe → Input (00)

Wenn Ausgabe → General purpose output mode (01)

Address offset: 0x00

Output type register (GPIOx_OTYPER)

Wenn konstant gleicher Value → Output open-drain

Wenn unterschiedliche Values → Output push-pull

Address offset: 0x04

Pull-up/pull-down register (GPIOx_PUPDR)

Oft No pull-up, no pull-down weil einfach ein Value gesendet wird oder bloss ein Value erhalten wird.

Address offset: 0x0C

Output speed register (GPIOx_OSPEEDR)

Wenn nichts definiert ist, dann Medium Speed.

Address offset: 0x08

GPIO port bit set/reset register (GPIOx_BSRR)

Wenn Port-Bits gecleart/gelöscht werden muss.

Address offset: 0x18

GPIO port output data register (GPIOx_ODR)

Liest den aktuellen Zustand des Outputs und gibt zurück. Kann auch geschrieben werden.

Address offset: 0x14

GPIO port input data register (GPIOx_IDR)

Liest den aktuellen Zustand des Inputs und gibt zurück.

Address offset: 0x10

Serial Data Transfer

SPI vs. I2C

SPI:

  • Master/Slave
  • Synchronous full-duplex transmission (MOSI, MISO)
  • Selection of device through Slave Select (SS)
  • No acknowledge, no error detection
  • Four modes → clock polarity and clock phase

I2C:

  • Synchronous half-duplex transmission (SCL, SDA)
  • 7-bit slave addresses

Synchronisation zwischen Sender und Empfänger bei UART

Durch die Benutzung der Start, Stopp und IDLE bits, kann man Sender und Empfänger synchronisieren.

Erstes Bits (Start) = 0 Fallende Flanke, danach Daten

Letztes Bit (Stopp, IDLE) = 1 Danach entweder eine Fallende Flanke wieder (Start Bit) um zu sagen, jetzt kommen weitere Daten. Oder konstant eine 1 als Value um zu signalisieren bleib IDLE, es kommen momentan keine weitere Daten.

Timers / Counters

Capture

Bei einem Event wird der Inhalt des Counter Registers in das Capture / Compare Register kopiert. Der Counter läuft weiter.

Compare

Sobald der Counter den Wert des Capture / Compare Register erreicht hat, wird ein Event oder ein Interrupt ausgelöst. Der Counter läuft weiter.

// Check Bit1 (EOC) of ADC1 status register.
// Address of status register of ADC1 register is:
// 0x4001'2000 + base offset of ADC1 + status reg offset = 0x4001'2000

#define MY_STATUS_REG_ADC1 (*((volatile uint8_t *)(0x40012000)))
// Solange 0x4001'2000. Gibt folgende Bit Operation 0 = False.
// 0x4001'2000 AND 0x02 = 0. Aber weil wir eine negation vorne haben gibt es True.
// !0 = True. Aus diesem Grund BLEIBT es in der While-Schleife und wartet.

// Solange es True ist, wird gewartet bis die Konvertierung zueende ist.
// Anschliessend gibt die obere Bit Operation folgendes Resultat:
// 0x4001'2002 AND 0x02 = 1. Aber weil wir eine negation vorne haben gibt es False.
// !0 = False. Aus diesem Grund FALLEN wir aus der While-Schleife.
while (!(MY_STATUS_REG_ADC1 & 0x02)){}

Asynchronous SRAM (statisches RAM)

  • Flip-flop basierte Struktur
  • Statisch: Kein Refresh nötig
  • Jeder Zugriff benötigt den gleichen Zeitaufwand

SDRAM (Synchronous Dynamic RAM)

  • Kondensator basierte Struktur
  • Periodischer Refresh nötig
  • Hohe Dichte
  • Synchrones Interface
  • Hohe Latenz am Anfang, späterer Zugriff wieder schnell
  • Block weiser transfer (eignet sich gut für Blockzugriffe)
  • Ist ein flüchtiger (volatiler) Speicher
  • Preis pro Speicherzelle ist niedrig

Principle of Locality

  • Spatial locality → Der aktuelle Speicherort der Daten liegt wahrscheinlich in der Nähe des nächsten Zugriffsortes.
  • Temporal locality → Auf den aktuellen Datenbestand wird wahrscheinlich in naher Zukunft erneut zugegriffen.
